# Apple II Software & ROM Database Search Page

* **GitHub Pages 라이브 데모**: [https://meesokim.github.io/apple2-database/](https://meesokim.github.io/apple2-database/)

이 프로젝트는 모든 Apple I, Apple II, Apple IIgs, Apple III 관련 XML 소프트웨어 목록 파일(`apple*`)을 통합하여 웹 브라우저에서 편리하게 검색하고 조회할 수 있는 반응형 웹 페이지입니다. Flickr 스크린샷 매핑(다운로드 및 로컬 캐싱), Google 이미지 검색 폴백, 유튜브 영상 사전 검색 연동 및 한글 번역이 내장되어 있습니다.

## 파일 구성

1. **`index.html`**: 메인 사용자 인터페이스 페이지 (반응형 디자인, 시스템 통계 대시보드, 다중 필터링, 정렬 기능 제공)
2. **`index.css`**: 현대적인 레트로-퓨처리스틱 다크 모드 스타일시트 (Apple 기종별 전용 시스템 배지 색상 적용)
3. **`index.js`**: 검색, 필터링, 한글 타이틀 노출, 페이지네이션, 모달 및 복사 기능 등 인터랙티브 로직
4. **`parse_apple2.py`**: 워크스페이스 내 XML 파일들을 읽고 `title_translations.json`의 한글 번역 데이터 및 `screenshot_map.json`을 대조하여 컴파일하는 빌드 스크립트
5. **`apple2_data.js`**: 파서를 통해 컴파일된 7,163개의 소프트웨어 및 한글 번역 통합 데이터
6. **`title_translations.json`**: 번역된 6,895개의 영문-한글 제목 매핑 캐시 파일
7. **`build_translations.py`**: Google 번역 API를 통해 전체 영문 타이틀을 한글로 일괄 번역하여 캐시를 채우는 자동 번역 스크립트
8. **`scrape_flickr.py`**: Jason Scott의 Flickr Apple II 스크린샷 앨범을 크롤링하여 사진 정보 목록을 수집하는 스크립트
9. **`flickr_screenshots.json`**: 수집된 Flickr 스크린샷 앨범 메타데이터 파일
10. **`download_screenshots.py`**: Flickr 앨범으로부터 각 소프트웨어의 인게임 스크린샷을 XML 데이터베이스와 매칭하여 다운로드받아 로컬 캐시 폴더에 보관해 주는 스크립트
11. **`screenshots/`**: 다운로드 완료된 인게임 이미지 파일들이 저장되는 로컬 디렉토리
12. **`screenshot_map.json`**: 다운로드된 이미지 파일과 데이터베이스 내 소프트웨어 제목 간의 매핑 캐시 파일
13. **`fetch_youtube_links.py`**: 유튜브에서 각 소프트웨어의 플레이 영상을 백그라운드에서 검색 및 크롤링하여 비디오 ID를 수집해주는 스크립트
14. **`youtube_links.json`**: 수집된 소프트웨어별 유튜브 비디오 ID 매핑 캐시 파일

## 사용 방법

### 1. 웹 페이지 실행하기
별도의 웹 서버 없이 브라우저로 `index.html` 파일을 직접 열어도 모든 기능이 완벽히 작동합니다. (CORS 보안 오류를 피하기 위해 스크립트 방식으로 데이터를 로딩합니다.)

* **직접 실행**: 파일 탐색기에서 `index.html` 파일을 더블 클릭하여 웹 브라우저로 엽니다.
* **로컬 웹 서버로 실행 (추천)**:
  터미널에서 아래 명령을 실행하여 로컬 서버를 구동한 뒤 브라우저로 접속합니다.
  ```bash
  python3 -m http.server 8000
  ```
  이후 브라우저에서 `http://localhost:8000` 주소로 접속하면 됩니다.

### 2. 데이터베이스, 번역 및 스크린샷 갱신하기 (Rebuild, Translate & Screenshots)
새로운 게임 파일이 추가되어 한글 번역 캐시, 스크린샷 이미지 및 데이터베이스를 새로 고치고 싶다면 아래 단계를 수행합니다.

1. **신규 번역 캐시 갱신**:
   ```bash
   python3 build_translations.py
   ```
2. **Flickr 스크린샷 목록 크롤링**:
   ```bash
   python3 scrape_flickr.py
   ```
3. **매칭 스크린샷 다운로드 및 맵 파일 갱신**:
   ```bash
   python3 download_screenshots.py
   ```
4. **유튜브 플레이 영상 ID 수집**:
   ```bash
   python3 fetch_youtube_links.py
   ```
5. **데이터베이스 최종 재컴파일**:
   ```bash
   python3 parse_apple2.py
   ```

## 주요 기능 안내

* **대시보드 통계**: Apple 사양별(Apple I, Apple II, Apple IIgs, Apple III) 소프트웨어 개수와 미디어 타입별(카세트, 플로피디스크, ROM 이미지 등) 보유 비율을 실시간 프로그레스 바 대시보드로 시각화합니다.
* **통합 검색 기능**: 소프트웨어명(Title)뿐만 아니라 고유 ID, 한글 타이틀, 시리얼 번호, 제작사(Publisher), 그리고 각 파트의 디스크/롬 파일명, CRC32 해시, SHA1 해시 정보까지 실시간으로 매칭하여 검색합니다.
* **그리드 & 리스트 뷰 전환**: 카드 형태의 예쁜 그리드 뷰와 콤팩트하고 대량 확인에 유용한 테이블 표(리스트) 형태의 뷰 토글 기능을 지원합니다.
* **고급 필터링**: 시스템 사양, 미디어 카테고리, 제작사, 출시연도 별 드롭다운 필터와 복제본(Clones) 제외 토글, 미지원 덤프 제외 토글을 제공합니다.
* **상세 보기 모달**: 게임 카드 혹은 리스트 줄의 '상세 보기'를 누르면 모달 창을 통해 해당 소프트웨어의 세부 메타데이터와 실제 디스크 파일들의 파트 구성, 각 파일 용량(KB/MB 변환), CRC32 해시, SHA1 해시, 덤프 상태를 볼 수 있습니다.
* **Flickr 스크린샷 매핑 & Google 이미지 검색 폴백**: 
  * Flickr 앨범으로부터 다운로드된 로컬 스크린샷이 존재할 경우 모달 상단에 자동으로 사진이 나타납니다.
  * 로컬에 스크린샷이 매핑되어 있지 않은 경우, 모달 상단 영역에 Google Image Search 바로가기 단추(**`Google 이미지에서 "<소프트웨어 제목>" 스크린샷 검색`**)가 생성되어 원터치로 구글에서 이미지를 찾아볼 수 있습니다.
* **유튜브 플레이 영상 연동**: 소프트웨어 상세 보기 내에 해당 소프트웨어의 유튜브 플레이 영상을 직접 재생할 수 있는 내장형 플레이어(Iframe Player)가 임베딩되어 보여집니다. 만약 매핑된 영상이 없는 경우 유튜브에서 원터치로 해당 소프트웨어의 동영상을 찾을 수 있는 '유튜브 검색' 링크가 제공됩니다.
* **해시 복사 단축 버튼**: 상세 보기 내 파일 목록에서 CRC32 및 SHA1 값 우측의 복사 아이콘을 누르면 클립보드에 해당 값이 원터치로 복사됩니다.
